# AstraNotes Working Agreement and Definition of Done

## Working Agreement

1. AI may be used as a development assistant, but all outputs must be reviewed before acceptance.
2. Project requirements will serve as the source of truth.
3. Work will be completed in small, manageable increments.
4. Major design and implementation decisions will be documented.
5. Features will be tested before being marked complete.
6. Unnecessary scope expansion will be avoided.
7. GitHub will be used to track project progress.
8. Simplicity, maintainability, and usability will be prioritized.

## Definition of Done

A feature is considered complete when:

1. The feature satisfies its related requirement.
2. The feature can be explained clearly.
3. Basic testing has been completed.
4. Security and privacy considerations have been reviewed.
5. Documentation has been updated if necessary.
6. The feature is ready for demonstration.
