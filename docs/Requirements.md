# AstraNotes Requirements

## Functional Requirements

FR-1: Users shall be able to create notes.

FR-2: Users shall be able to edit notes.

FR-3: Users shall be able to delete notes.

FR-4: Users shall be able to view saved notes.

FR-5: Users shall be able to search notes by keyword.

FR-6: Users shall be able to save notes locally using SQLite.

FR-7: Users shall be able to organize notes using tags.

FR-8: The system shall support Markdown formatting in note content.

## Non-Functional Requirements

NFR-1: The application shall provide a simple and easy-to-use interface.

NFR-2: The application shall load and save notes within a reasonable response time.

## Security and Reliability Requirements

SR-1: Note content shall be encrypted before storage.

SR-2: User data shall remain private on the local device.

SR-3: User input shall be validated before storage.
